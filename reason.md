# 활성 파일이 없을 때 빈 목록이 표시되지 않는 이유

## 문제 상황
모든 탭이 빈 탭일 때 (활성 파일이 없을 때) 빈 목록이 표시되지 않음

## 원인 분석

### 1. file-open 이벤트가 발생하지 않음
```
file-open 이벤트 리스너:
- if (file) { this.handleFileChange(file, false); }
```

**문제점:**
- 모든 탭을 닫거나 빈 탭으로 전환할 때 `file-open` 이벤트는 `file`이 `null`로 발생함
- 하지만 코드는 `if (file)` 조건으로 `null`인 경우를 무시함
- 따라서 빈 탭으로 전환되어도 뷰가 업데이트되지 않음

### 2. 뷰가 업데이트되지 않는 흐름

**현재 상황:**
1. 사용자가 파일을 열고 있음 → 목록 표시됨
2. 사용자가 모든 탭을 닫음 → `file-open` 이벤트 발생 (file = null)
3. `if (file)` 조건에 걸려서 `handleFileChange` 호출 안 됨
4. 뷰는 여전히 이전 파일 목록을 표시하고 있음
5. 빈 목록으로 업데이트되지 않음

### 3. 해결 방법

**file-open 이벤트 리스너 수정 필요:**
```
현재: if (file) { ... }
수정: file이 null일 때도 뷰 업데이트 호출
```

**세부 방안:**
- `file`이 `null`일 때도 뷰의 `updateView()`를 직접 호출
- 또는 `handleFileChange` 로직을 수정하여 `null` 파일도 처리
- 또는 모든 뷰를 순회하면서 `updateView()` 호출

## 요약
활성 파일이 없을 때(빈 탭일 때) `file-open` 이벤트의 `file` 파라미터가 `null`이 되는데, 현재 코드는 `if (file)` 조건으로 `null`을 무시하기 때문에 뷰가 업데이트되지 않음. 따라서 이전에 표시된 파일 목록이 그대로 남아있게 됨.
